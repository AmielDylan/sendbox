# Sendbox V1 - Studio créatif

Ce dossier sert de zone de travail pour les propositions visuelles V1.

## Règle de collaboration

Le studio propose. Le cockpit intègre.

Dans un échange dédié avec un modèle orienté UI/UX ou génération d'images, les sorties doivent rester ici :

- `docs/design/studio/` pour les briefs, audits, propositions, prompts et décisions.
- `public/images/studio/v1/` pour les images générées ou variantes candidates.

Le studio ne modifie pas directement :

- `app/`
- `components/`
- `lib/`
- `hooks/`
- `styles/`

Les intégrations dans l'application se font ensuite depuis le cockpit projet, dans une branche dédiée, après sélection.

## Objectif V1

Faire passer Sendbox d'une interface fonctionnelle mais encore moyenne à une expérience plus professionnelle, rassurante et engageante, sans alourdir la V1.

France-Bénin est le premier corridor de lancement. La direction visuelle doit toutefois rester pensée pour une plateforme multi-pays à terme, capable d'ouvrir d'autres corridors dans les deux sens.

Le rendu attendu doit être :

- épuré, mais pas pauvre ;
- professionnel, mais pas froid ;
- naturel, humain et crédible ;
- familier dans ses conventions d'interface ;
- cohérent avec une plateforme de confiance, de mise en relation et de colis entre particuliers.

## Contraintes produit

Sendbox V1 :

- sécurise la mise en relation entre expéditeur et voyageur vérifiés ;
- facture uniquement les frais Sendbox de mise en relation : 2,90 EUR ;
- ne facture pas le transport sur la plateforme ;
- ne reverse pas le voyageur ;
- ne vend pas d'assurance colis ;
- s'appuie sur KYC, déclaration colis, preuves photo, messages, avis, signalements et litiges.

Toute proposition visuelle doit éviter de promettre :

- une assurance active ;
- un transport garanti par Sendbox ;
- un escrow ou paiement du transport ;
- un remboursement automatique ;
- une couverture financière en cas de perte ou dommage.

## Fichiers de travail

- `v1-creative-brief.md` : contexte et direction créative.
- `v1-ui-audit.md` : audit de l'interface actuelle et corrections prioritaires.
- `v1-visual-direction.md` : direction artistique, tokens et règles de composants.
- `v1-landing-visual-proposals.md` : propositions d'images pour les sections de landing.
- `v1-image-prompts.md` : prompts de génération image.
- `v1-asset-manifest.md` : inventaire des assets attendus.
- `v1-ui-review-template.md` : grille d'audit UI pour les propositions.
