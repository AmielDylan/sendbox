# Sendbox V1 - Grille de revue UI

Utiliser cette grille pour auditer les propositions du studio avant intégration.

## Diagnostic rapide

Pour chaque écran, noter :

- lisibilité ;
- confiance ;
- hiérarchie visuelle ;
- cohérence des couleurs ;
- familiarité des patterns ;
- densité ;
- responsive mobile ;
- accessibilité visible ;
- promesse produit correcte.

Score simple :

- 1 = bloque la V1 ;
- 2 = acceptable mais faible ;
- 3 = solide pour V1 ;
- 4 = très bon ;
- 5 = excellent.

## Points à regarder

### Boutons

- Le bouton primaire est évident.
- Le secondaire est différent sans être faible.
- Les états hover/focus/désactivé sont cohérents.
- Les boutons ne mélangent pas trop d'icônes et de texte.

### Badges

- KYC, avis, statut colis et alertes ont chacun une logique visuelle stable.
- Les badges ne ressemblent pas à de la décoration gratuite.
- Les couleurs système gardent la même signification partout.

### Alertes

- Succès, information, attention, danger sont distincts.
- Le texte explique l'action suivante.
- Les couleurs ne crient pas inutilement.

### Cartes

- Les cartes servent à regrouper des informations répétées ou un état.
- Pas de cartes imbriquées.
- Rayons modérés, bordures fines, ombres discrètes ou absentes.

### Empty states

- L'image aide à comprendre l'état.
- Le titre est concret.
- Le CTA donne la prochaine action.
- L'état vide ne ressemble pas à une erreur.

### Landing

- Le hero explique Sendbox en moins de 10 secondes.
- L'image montre le vrai sujet : remise colis + confiance.
- Le tarif 2,90 EUR est visible sans paraître agressif.
- Les sections ne surpromettent pas assurance, escrow ou transport garanti.

### Mobile

- Les éléments clés restent visibles sans scroll excessif.
- Les tables deviennent cartes ou listes.
- Les images ne cachent pas les acteurs principaux.
- Les CTA restent accessibles.

## Sortie attendue

Le studio doit produire :

- une liste des 10 corrections UI les plus rentables pour la V1 ;
- les 3 corrections à ne pas faire maintenant ;
- les assets à intégrer en premier ;
- les risques de confusion produit à corriger.
